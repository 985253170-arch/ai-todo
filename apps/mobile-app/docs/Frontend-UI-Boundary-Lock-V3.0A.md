## 1. 产品定位锁定

清行是手机端优先的 AI 行动教练，不是普通 Todo List，不是聊天软件，也不是强 SaaS 工具。

核心闭环：

```txt
目标 → AI 拆解 → 执行 → 记录 → 复盘 → 智能调整
```

清行的核心体验是：

1. 帮用户把模糊目标变成今天能做的一小步。
2. 降低开始行动的压力。
3. 不替用户完成任务，只陪用户推进、验收、调整。
4. 每个界面只突出一个主动作。
5. 手机端优先，不能把网页功能堆成一个长页面。

---

## 2. 本文档解决的问题

本次 Batch 3 出错原因：

```txt
今日页中同时堆入：
目标概览 / 进度 / 当前任务 / 后续任务 / 执行入口
```

这导致：

1. 一个界面装不下。
2. 页面像普通 Todo List。
3. 当前任务不够聚焦。
4. 和设计稿中的三层页面结构不一致。

因此，后续所有前端实现必须按本文档锁定边界。

## 4. 全局视觉风格锁定

所有手机 App 页面必须保持以下风格：

| 维度 | 锁定规则 |
|---|---|
| 背景 | 暖米白、纸张感，不用纯白冷冰背景 |
| 主色 | 深蓝为主，不使用强紫色作为主视觉 |
| 辅色 | 暖黄、浅米、浅灰、淡棕，可少量点缀 |
| 卡片 | 纸张卡片感，大圆角，柔和阴影 |
| 图标 | 手绘风 SVG，不使用普通图标库 |
| 字体气质 | 温柔、安静、低压力 |
| 布局 | 手机端优先，375–430px 宽度优先 |
| 信息密度 | 单屏只突出一个主动作 |
| 导航 | 登录后底部 Tab 固定 |
| 禁止 | 强 SaaS 风、表格、密集列表、复杂仪表盘 |

## 5. 底部 Tab 锁定

登录后底部 Tab 固定为：

```txt
今日 / 足迹 / 成长 / 我的
```

禁止：

1. 改名。
2. 新增聊天 Tab。
3. 新增复杂项目管理入口。
4. 在任务执行界面隐藏底部 Tab。
5. 在 Auth 页面显示底部 Tab。

显示规则：

| 状态 | 是否显示底部 Tab |
|---|---|
| 产品首页 | 不显示 |
| 登录页 / 注册页 | 不显示 |
| 今日首页 | 显示 |
| 任务界面 | 显示 |
| 任务执行界面 | 显示 |
| 足迹页 | 显示 |
| 成长页 | 显示 |
| 我的页 | 显示 |

## 6. 页面层级总览

清行手机 App 不能把所有功能放在一个页面内。

页面必须分层：

```txt
产品首页
  ↓ 开始使用
登录 / 注册页
  ↓ 登录成功
今日首页
  ↓ 输入目标并生成任务
任务界面
  ↓ 点击当前任务
任务执行界面
  ↓ 退出 / 完成
任务界面
```

底部 Tab 内页面：

```txt
今日
  ├─ 今日首页
  ├─ 任务界面
  └─ 任务执行界面

足迹
  ├─ 足迹空状态页
  └─ 足迹历史页

成长
  └─ 洞察成长页

我的
  └─ 设置 / 我的页
```

## 7. 页面一：产品首页边界

### 7.1 页面职责

产品首页只负责：

1. 展示品牌。
2. 说明产品气质。
3. 引导用户开始使用或登录。

### 7.2 页面结构

必须包含：

1. 品牌名：清行。
2. 登录入口。
3. 温柔主标题，例如：

```txt
慢一点，
也在向前走
```

4. 副文案，例如：

```txt
不用完整计划，写下一句话，我会与你一起完成。
```

5. 产品预览卡片。
6. “行动中”贴纸或轻量装饰。
7. 主按钮：开始使用。
8. 次入口：已有账号，去登录。

### 7.3 跳转规则

| 操作 | 跳转 |
|---|---|
| 点击“开始使用” | 登录页，默认验证码登录 |
| 点击“已有账号，去登录” | 登录页，默认验证码登录 |
| 点击右上角“登录” | 登录页，默认验证码登录 |

### 7.4 禁止事项

1. 不显示底部 Tab。
2. 不显示任务列表。
3. 不显示统计数据。
4. 不进入注册页作为默认入口。
5. 不使用 `AI Todo`、`Handy Planner` 作为用户可见品牌。

## 9. 页面三：今日首页边界

### 9.1 页面职责

今日首页只负责：

1. 欢迎用户。
2. 让用户输入今天想推进的目标。
3. 点击生成任务。
4. 生成成功后自动进入“任务界面”。

今日首页不是任务列表页。

### 9.2 页面结构

必须包含：

1. 温柔问候：

```txt
早上好，今天我们一起向前一点点
```

2. 主标题：

```txt
今天，想完成哪一小步？
```

3. 手绘装饰，例如纸飞机、星星、叶子。
4. 目标输入卡片。
5. 输入 placeholder：

```txt
例如：准备明天的面试
```

```txt
不用想清全部，先写下一句话。
```

7. 主按钮：

```txt
迈出全新的一步
```

8. 轻量进度提示，例如：


## 10. 页面四：任务界面边界

### 10.1 页面职责

任务界面用于展示 AI 生成后的任务清单。

它是“任务清单页”，不是今日首页，也不是任务执行页。

### 10.2 页面进入方式

只允许从今日首页生成任务后进入：

```txt
今日首页 → 任务界面
```

也可以在已有今日任务时，从今日 Tab 进入任务界面，但不能回到堆叠型今日页。

### 10.3 页面结构

必须包含：

1. 标题：

```txt
已经开始了，今天慢慢来
```

2. 副标题：

```txt
不用看完整清单，先把眼前这一小步做好。
```

3. 今日目标摘要卡：

```txt
今天的小目标
准备明天的数据分析面试
已经拆好了，我们一步一步来。
```

4. 今日进度卡：

```txt
今天走到这里
已完成 0 / 4 小步
先完成第一步就好。
```

5. 当前任务卡。
6. 后续任务弱化列表。
7. 底部 Tab。

### 10.4 当前任务卡边界

当前任务卡必须是页面中最突出的主动作。

必须包含：

1. 标签：

```txt
先做这一件
```

2. 当前任务标题。
3. 任务细节，例如：

```txt
圈出 3 个关键词
约 10 分钟
```

4. 主按钮：

```txt
陪我做这一步
```

5. 次按钮：

```txt
我完成了
```

### 10.5 后续任务列表边界

后续任务必须弱化展示。

必须遵守：

1. 标题为：后面再做。
2. 只展示当前任务之后的任务。
3. 后续任务不能抢主动作。
4. 后续任务点击“陪我”不能进入执行页。
5. locked 任务点击后只提示：

```txt
先完成眼前这一小步
```

### 10.6 任务顺序执行规则

任务必须按顺序执行：

```txt
第一个未完成任务 = current
current 任务可进入执行界面
locked 任务不可进入执行界面
完成 current 后，下一个任务变 current
```

### 10.7 任务界面禁止事项

1. 不显示目标输入框。
2. 不显示任务执行反馈输入框。
3. 不显示 AI 步骤卡。
4. 不做聊天气泡。
5. 不做密集 checkbox Todo List。
6. 不用删除线表达完成。
7. 不做复杂项目管理。

## 11. 页面五：任务执行界面边界

### 11.1 页面职责

任务执行界面只负责陪用户完成当前这一小步。

它不是任务列表页，也不是全局聊天页。

### 11.2 进入方式

只允许从任务界面的 current 任务进入：

```txt
任务界面 → 点击 current 任务 / 陪我做这一步 → 任务执行界面
```

locked 任务不能进入任务执行界面。

### 11.3 页面结构

必须包含：

1. 顶部标题：

```txt
陪你走这一步
```

2. 右侧退出按钮：

```txt
先退出
```

3. 当前任务卡：

```txt
现在这件事
准备 1 分钟自我介绍
完成要不要打勾，最后由你决定。
```

4. AI 步骤卡：

```txt
先做这一小步
- 写完这三行。
- 写完这三行。
- 写完这三行。

写完这三行，回来告诉我你写到哪了。
```

5. 反馈输入区：

```txt
把现在的情况告诉我
```

6. textarea placeholder：

```txt
写下你做到哪了、卡在哪里，或贴一小段草稿。
```

7. 按钮：

```txt
发给 AI
```

8. 轻提示：

```txt
不用写很多，一句话也可以
```

9. 底部 Tab 保留。

### 11.4 任务执行界面交互

| 操作 | 结果 |
|---|---|
| 点击“先退出” | 回到任务界面 |
| 输入框为空 | “发给 AI”禁用 |
| 点击“发给 AI” | 替换当前 AI 步骤卡内容，清空输入框 |
| 点击底部 Tab | 可切换到其他 Tab |
| 从其他 Tab 返回今日 | 保持合理的今日状态 |

### 11.5 AI 陪伴边界

必须遵守：

1. AI 陪伴绑定当前任务。
2. AI 只推进当前一步。
3. AI 可以给材料、框架、清单、低风险草稿片段。
4. AI 不替用户完成最终结果。
5. AI 不自动勾选任务。
6. 完成权在用户手中。

### 11.6 禁止做成聊天软件

任务执行界面禁止：

1. 聊天气泡 UI。
2. 左右对话布局。
3. 消息历史列表。
4. 使用 `messages.map()` 渲染聊天记录。
5. 全局聊天入口。
6. 聊天 Tab。
7. 长期对话记忆。
8. 脱离当前任务的自由聊天。

## 15. 状态机边界锁定

### 15.1 Auth 状态

```ts
type AuthState = "guest" | "authenticated";
```

规则：

```txt
guest → 产品首页 / 登录 / 注册
authenticated → AppShell + 底部 Tab
```

### 15.2 今日 Tab 内状态

建议锁定为：

```ts
type TodayMode = "home" | "tasks" | "companion";
```

含义：

| TodayMode | 页面 |
|---|---|
| home | 今日首页，只输入目标和生成任务 |
| tasks | 任务界面，展示生成后的任务列表 |
| companion | 任务执行界面，只执行当前任务 |

禁止继续使用模糊命名导致误解，例如：

```ts
"empty" | "tasks" | "companion"
```

因为 `empty` 容易被误解成“没有任务时的今日页”，但实际今日首页也承担目标输入和开始行动的职责。推荐使用 `home`。

### 15.3 页面切换规则

```txt
Auth guest:
  product-home → otp-login → password-login / register

Auth authenticated:
  今日 Tab:
    home → generateTasks → tasks
    tasks → click current task → companion
    companion → 先退出 → tasks

  足迹 Tab:
    no-history → footprint-empty
    has-history → footprint-history

  成长 Tab:
    growth

  我的 Tab:
    me
```
