# V3.0A 手机视口与高度预算锁定文档

> 状态：手机视口硬约束锁定  
> 适用范围：`apps/mobile-app/` 手机 App 前端界面  
> 目的：防止页面做成长网页，确保每个页面在 390×844 手机视口内可用  
> 优先级：与 `UI-Spec-Design-Screenshot-Lock-V3.0A.md` 同级，高于普通架构文档

---

## 1. 文档定位

本文件专门锁定 `apps/mobile-app` 的手机视口、高度预算、滚动边界、截图验收标准。

本文件**不是**视觉风格文档。视觉风格由 `UI-Spec-Design-Screenshot-Lock-V3.0A.md` 锁定。本文件是**手机 App 布局硬约束**文档——它回答的问题是：

```txt
这个页面在 390×844 的手机屏幕上，能不能一屏装下？
如果不能，哪里出了问题？
```

### 1.1 适用范围

后续所有 `apps/mobile-app` 前端 UI 实现、Review、视觉验收，必须读取本文件。

### 1.2 优先级

```txt
1. UI-Spec-Design-Screenshot-Lock-V3.0A.md       （最高：页面结构 + 视觉）
2. Mobile-Viewport-Layout-Lock-V3.0A.md           （最高：视口 + 高度预算）
3. Frontend-UI-Boundary-Lock-V3.0A.md             （边界 + 状态机）
4. 其他架构 / 执行方案文档
```

如果本文件与普通架构文档冲突，以本文件和 `UI-Spec-Design-Screenshot-Lock-V3.0A.md` 为最高优先级。

## 2. 目标视口

### 2.1 验收视口

```txt
主验收视口：390 × 844 （iPhone 14 Pro 逻辑分辨率）
兼容范围：375 × 812 到 430 × 932
```

所有页面必须优先保证 **390 × 844** 下体验成立。兼容范围是保证不崩，但首要验收标准是 390×844。

### 2.2 核心原则

```txt
清行是手机 App，不是网页长页。
手机 App 页面默认不应依赖整页长滚动。
每一屏只做一件事。
当前任务优先。
底部 Tab 必须首屏可见。
```

## 3. 全局布局模型

### 3.1 推荐 App 布局模型

```txt
┌──────────────────────────┐
│  AppShell               │  h-screen, flex flex-col, overflow-hidden
│  ┌────────────────────┐ │
│  │  内容区            │ │  flex-1, min-h-0, overflow-hidden
│  │  ┌──────────────┐  │ │
│  │  │ 固定区域      │  │ │  header / 核心卡片（不滚动）
│  │  ├──────────────┤  │ │
│  │  │ 可滚动区域    │  │ │  overflow-y-auto, max-h 明确
│  │  └──────────────┘  │ │
│  └────────────────────┘ │
│  BottomTabBar           │  fixed / 固定高度 ~76px
└──────────────────────────┘
```

关键 CSS 模式：

```txt
AppShell 外层：   h-screen + flex flex-col + overflow-hidden
内容区：          flex-1 + min-h-0 + overflow-hidden
可滚动的子区域：  max-h-[...] + overflow-y-auto
底部 Tab：        fixed bottom-0 或 flex-shrink-0
```

### 3.2 明确禁止

```txt
❌ 不受控的 min-h-screen 长页思路
❌ min-h-[calc(100vh-...)] 撑高页面（"至少一屏高，上不封顶"）
❌ 无 max-height 的纵向卡片堆叠
❌ 让整个页面靠滚动才能看到核心内容
❌ 底部 Tab 被内容挤出或需要滚动才能看到
❌ 用 min-h 代替 h 作为外层容器
❌ 内容区不设 overflow-hidden，任由子元素撑开
❌ flex-col 子元素不配合 flex-1 + min-h-0
```

### 3.3 正确写法示例

```tsx
// ✅ 正确：AppShell 约束视口
<div className="h-screen flex flex-col overflow-hidden">
  <main className="flex-1 min-h-0 overflow-hidden px-6 pt-8">
    {children}
  </main>
  <BottomTabBar />
</div>

// ✅ 正确：页面内容在 flex-1 容器内
<div className="flex flex-col h-full gap-4">
  <header>{/* 固定高度 */}</header>
  <div className="flex-1 min-h-0">{/* 弹性区域 */}</div>
  <div className="overflow-y-auto max-h-[180px]">{/* 可滚动列表 */}</div>
</div>

// ❌ 错误：min-h-screen 允许内容撑高
<div className="min-h-screen">
  {/* 内容多了就变长页 */}
</div>

// ❌ 错误：min-h-[calc(100vh-7rem)] 不设上限
<div className="flex min-h-[calc(100vh-7rem)] flex-col gap-5">
  {/* 内容无限堆叠 */}
</div>
```

## 4. 滚动边界规则

### 4.1 逐页默认规则

| 页面 | 整页滚动 | 内部滚动 | 备注 |
|------|---------|---------|------|
| 今日首页 | ❌ 不允许 | 不需要 | 内容少，天然一屏 |
| 任务界面 | ❌ 不允许 | UpcomingTaskList 区域可 `overflow-y-auto` | 如果内容放不下，优先压缩卡片，其次让"后面再做"内部滚动 |
| 任务执行界面 | ❌ 默认不允许 | 反馈输入区可内部适配 | 首屏核心内容必须可见 |
| 足迹空状态页 | ❌ 不允许 | 不需要 | 内容少 |
| 足迹历史页 | ❌ 不允许 | 历史列表区域 `overflow-y-auto` | 列表可内部滚动 |
| 成长页 | ⚠️ 允许轻微 | 建议卡片区 `overflow-y-auto` | 首屏必须完整展示核心卡片 |
| 我的页 | ❌ 不允许 | 设置列表区域 `overflow-y-auto` | 列表可内部滚动 |

### 4.2 任务界面专项规则

```txt
任务界面如果内容放不下：
1. 优先压缩卡片高度
2. 其次让"后面再做"区域内部滚动（max-h + overflow-y-auto）
3. 不允许让整个任务界面变成长网页
4. 不允许压缩底部 Tab
5. 不允许隐藏"后面再做"标题

如果 390×844 下内容确实无法一屏容纳：
→ 仅 UpcomingTaskList 区域设 max-h + overflow-y-auto
→ 其余部分保持固定
```

## 5. 任务界面高度预算（390 × 844）

### 5.1 高度预算表

```
┌──────────────────────────────┬─────────┬─────────┐
│ 区域                         │ 高度    │ 累计    │
├──────────────────────────────┼─────────┼─────────┤
│ AppShell 顶部留白 (pt-8)     │ 32px    │ 32px    │
│ 退出栏 + 任务执行标题         │ 32–40px │ 64–72px │
│ 标题"已经开始了，今天慢慢来"   │         │         │
│ + 副标题 + 星星装饰           │ 86–110px│ 150–182px│
│ TaskProgressCard (2 卡片)    │ 120–145px│ 270–327px│
│ 区域间 gap 小计 (×4)         │ 48–60px │ 318–387px│
│ CurrentTaskCard              │ 220–260px│ 538–647px│
│ UpcomingTaskList             │ 140–180px│ 678–827px│
│ AppShell 底部留白 (pb)       │ —       │ —       │
│ BottomTabBar                 │ 72–84px │ —       │
├──────────────────────────────┼─────────┼─────────┤
│ 内容区总高                    │ ~827px  │         │
│ 可用视口 (含 Tab)             │ 844px   │         │
│ 内容区可用 (844 - 32 - 76)   │ ~736px  │         │
│ 超出 / 剩余                   │ ~91px   │超出！    │
└──────────────────────────────┴─────────┴─────────┘

注意：内容区总高 ~827px 已超过可用高度 ~736px。
必须采用以下策略之一：
A. 压缩卡片高度，使总高降至 ~736px
B. UpcomingTaskList 设 max-h-[180px] overflow-y-auto
```

### 5.2 首屏必须可见清单（P0 标准）

390×844 视口下，任务界面首屏必须看到以下**全部**内容：

```txt
☑ ← 退出 / 任务执行
☑ 已经开始了，今天慢慢来
☑ 副标题：不用看完整清单，先把眼前这一小步做好。
☑ 今天的小目标卡片
☑ 今天走到这里卡片（含进度条）
☑ 当前任务黄色卡片（标签 + 标题 + 细节 + 按钮）
☑ "后面再做"标题 + 至少 2 条后续任务
☑ 底部 Tab（今日激活）
```

如果 390×844 下看不到以上任何一项，视为 **P0**。

### 5.3 各模块尺寸建议

```txt
TaskProgressCard：
  - 两张卡片合并为一张或紧凑排列
  - 单卡 max-height: ~145px
  - padding 使用 "compact" 而非 "normal"

CurrentTaskCard：
  - 标签行: ~28px
  - 标题: ~48px
  - 细节: ~36px
  - 按钮区: ~96px
  - 合计: ~220-260px
  - 不允许超过 280px

UpcomingTaskList：
  - 标题 "后面再做": ~32px
  - 每项: ~52px (含 padding)
  - 3 项: ~156px + gap
  - 容器 max-h: 180px, overflow-y-auto
  - 超过 3 项时内部滚动
```

## 8. 禁止布局模式

### 8.1 P0 禁止项（出现即视为 Review 不通过）

```txt
1. 任务界面用整页长滚动承载所有模块
2. 页面内容总高度超过 390×844 视口可用高度 20% 以上
3. 底部 Tab 需要滚动才能看见
4. CurrentTaskCard 或 UpcomingTaskList 高度无限撑开
5. 任务界面像网页长页（连续滚动才能看完所有内容）
6. 任务界面像普通 Todo List（密集勾选 + 删除线）
7. 用 lint/build 通过代替手机截图验收
8. AppShell 内容区使用 min-h-screen 导致无上限
9. 页面外层使用 min-h-[calc(100vh-...)] 不设上限
10. flex-col 内容区不加 flex-1 + min-h-0
```

### 8.2 P1 禁止项（出现即必须修复）

```txt
1. 卡片 padding 使用 "large" 导致高度显著膨胀
2. gap 过大导致不必要的间距累积
3. 后续任务列表不设 max-h，撑开整页
4. 装饰元素挤占过多首屏空间
5. 卡片独立成多张导致总高度超出预算
```

## 9. Review 截图验收流程

### 9.1 Claude Code Review 必须执行的验收步骤

```txt
1. 打开浏览器 DevTools
2. 设置 Responsive Mode，选择 390 × 844（iPhone 14 Pro）
3. 导航到目标页面
4. 截图整页（含底部 Tab）
5. 对照 UI-Spec-Design-Screenshot-Lock-V3.0A.md 设计稿
6. 逐区标注高度，对照本文档预算表
7. 检查是否需要整页滚动才能看到所有核心模块
8. 检查底部 Tab 是否首屏可见
9. 检查"后面再做"是否在首屏可见
10. 如果内容超出视口，判断是哪个区域超出，给出压缩建议
```

### 9.2 验收判定

```txt
lint 通过 ≠ 视觉通过
build 通过 ≠ 视觉通过
模块存在 ≠ 一屏容纳

只有 390×844 截图中全部核心模块可见 → 视觉通过
```

### 9.3 截图检查清单

| 检查项 | 方法 |
|--------|------|
| 底部 Tab 首屏可见 | 截图底部是否显示 Tab |
| 核心卡片不超出视口 | 截图是否包含所有核心模块 |
| 不需要整页滚动 | DevTools 中页面是否可一屏展示 |
| 间距与设计稿一致 | 截图与设计稿逐区对比 |
| 没有禁止项出现 | 对照本文档 §8 |

