## 2. 产品信息架构

### 2.1 顶层结构

手机 App 使用四个底部 Tab，已锁定不可随意更改：

```txt
今日 / 足迹 / 成长 / 我的
```

| Tab | 设计稿 | 功能定位 |
|---|---|---|
| 今日 | `主页.png`、`任务页面.png`、`任务执行界面.png` | 输入目标、生成任务、执行当前任务 |
| 足迹 | `足迹.png`、`足迹历史页.png` | 查看历史行动记录 |
| 成长 | `洞察成长页.png` | 查看完成率、连续天数、AI 建议 |
| 我的 | `设置我·的页.png` | 账号、同步、隐私、反馈、版本、退出 |

### 2.2 页面流转总览

```txt
未登录用户
  ↓
产品欢迎页
  ├─ 开始使用 → 注册页 / 验证码登录页
  └─ 已有账号，去登录 → 验证码登录页

验证码登录页
  ├─ 验证码登录
  ├─ 密码登录 Tab → 密码登录页
  └─ 创建我的行动记录 → 注册页

注册页
  ├─ 输入邮箱
  ├─ 输入 6 位验证码
  ├─ 设置密码
  ├─ 再次输入密码
  └─ 开始我的行动记录 → 今日首页

登录成功
  ↓
今日 Tab
  ├─ 空任务状态 → 输入目标 → 迈出全新的一步
  ├─ 已生成任务状态 → 陪我做这一步 / 我完成了
  └─ 任务执行陪伴页 → 发给 AI / 返回今日

底部 Tab
  ├─ 今日
  ├─ 足迹
  ├─ 成长
  └─ 我的
```

## 3A. 品牌命名规范

### 3A.1 对外品牌名

```txt
清行
```

含义方向：

```txt
清爽、轻盈、安静地行动。
不强调复杂计划，不制造压力，只陪用户完成今天这一小步。
```

### 3A.2 内部项目代号

```txt
AI Todo
```

内部代码、历史文档、仓库名可以继续保留 AI Todo。  
但所有新手机 App UI 的对外展示名称必须统一为 `清行`。

### 3A.3 品牌主文案

```txt
慢一点，也在向前走
```

### 3A.4 商标风险备注

`清行` 当前仅作为 V3.0A 原型品牌名使用。正式上线、App Store 发布、商业推广或商标注册前，必须进行正式商标检索与法务确认。

| 次级文字 | `#7A756B` / `#8A8278` | 辅助说明 |

### 3.3 字体与字重

| 用途 | 样式 |
|---|---|
| App 标题 / 页面大标题 | 中文衬线感或加粗宋黑风格，深蓝，字号 36–48 |
| 英文 Logo | 斜体 / Serif 风格，深蓝或紫蓝 |
| 卡片标题 | 加粗，24–32 |
| 正文 | 16–22 |
| 辅助说明 | 14–18 |
| 按钮文字 | 18–24，加粗 |

如果工程暂时不接入特殊字体，使用系统字体兜底：

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif;
```

标题可单独使用：

```css
font-family: Georgia, "Songti SC", "STSong", serif;
```

### 3.4 圆角与阴影

| 元素 | 圆角 | 阴影 |
|---|---:|---|
| 大卡片 | 28–36px | 柔和大阴影 |

| 主按钮 | 999px | 底部轻阴影 |
| 小标签 | 999px | 无或轻阴影 |

| 项目 | 规格 |

## 4. 全局组件规范

### 4.1 AppShell

所有已登录主页面共用一层 AppShell。

```txt
AppShell
├─ 当前 Tab 页面内容
└─ BottomTabBar
```

约束：

- AppShell 只能有一层，不允许出现“双重界面”。
- 任务执行页仍属于 AppShell 内的当前任务执行层，不允许再套一层完整 App。
- 底部 Tab 固定在底部。
- 页面内容需要给底部导航预留 padding-bottom。

### 4.2 BottomTabBar

四个 Tab：

```txt
今日 / 足迹 / 成长 / 我的
```

| Tab | 图标建议 | 激活色 |
|---|---|---|
| 今日 | 日历 / 指南针 / 首页 | 深蓝 |
| 足迹 | 脚印 | 深蓝 |
| 成长 | 柱状图 / 小芽 / 上升线 | 深蓝 |
| 我的 | 用户头像 | 深蓝 |

规则：

- 激活 Tab 图标和文字为深蓝。
- 未激活 Tab 图标和文字为灰褐色。
- 点击 Tab 切换页面，不刷新，不调用真实接口。
- 当前阶段用 mock 数据。

### 4.3 PaperCard

用于白色纸张卡片。

通用样式：

```txt
背景：暖白
圆角：28–36px
边框：浅米色
阴影：柔和
内边距：24–32px
```

### 4.4 PrimaryButton

主按钮样式：

```txt
深蓝背景
白色文字
圆角 999px
高度 56–68px
轻微阴影
```

常见文案：

- 开始使用
- 进入我的行动手账
- 开始我的行动记录
- 迈出全新的一步
- 陪我做这一步
- 看看下次怎么开始
- 去完成今天这一小步
- 发给 AI

### 4.5 SecondaryButton

次按钮样式：

```txt
白色 / 米白背景
深蓝文字
圆角 999px
浅边框
```

常见文案：

- 我完成了
- 直接回来
- 已有账号，去登录
- 看看那天做了什么

### 4.6 TextInput

输入框样式：

```txt
圆角胶囊
浅蓝灰边框
左侧图标
placeholder 灰色
高度约 60px
```

输入框类型：

- 邮箱地址
- 6 位验证码
- 设置密码
- 再次输入密码
- 今天想完成的小步
- 当前情况反馈 textarea

## 10. 页面规格：今日任务页 / 已生成任务状态

### 10.1 对应设计稿

```txt
任务页面.png
```

### 10.2 页面目标

展示今天的目标、完成进度、当前最应该做的一件事，并弱化后续任务。

### 10.3 页面结构

```txt
顶部：
- 大标题：已经开始了，今天慢慢来
- 副标题：不用看完整清单，先把眼前这一小步做好。
- 星星装饰

中部总卡片：
- 小卡 1：今天的小目标
  - 准备明天的数据分析面试
  - 已经拆好了，我们一步一步来。
- 小卡 2：今天走到这里
  - 已完成 0 / 4 小步
  - 进度条
  - 先完成第一步就好。

当前任务强调卡：
- 标题：先做这一件
- 任务标题：阅读面试岗位要求
- 补充信息：
  - 圈出 3 个关键词
  - 约 10 分钟
- 主按钮：陪我做这一步
- 次按钮：我完成了

后续任务卡：
- 标题：后面再做
- 任务列表：
  - 准备 1 分钟自我介绍 · 15 分钟
  - 整理简历项目经历 · 20 分钟
  - 复习常见 SQL 问题 · 15 分钟
- 每项右侧：陪我
- 圆形未完成状态
```

### 10.4 交互

| 元素 | 行为 |
|---|---|
| 陪我做这一步 | 进入任务执行界面 |
| 我完成了 | 当前任务标记完成，进度 +1，下一任务变成当前任务 |
| 后续任务“陪我” | 如果任务 locked，点击提示“先完成眼前这一小步” |
| 底部 今日 Tab | 保持当前页 |

### 10.5 任务状态

| 状态 | UI |
|---|---|
| current | 黄色强调卡，大按钮 |
| locked | 后续任务列表，弱化，不能直接执行 |
| completed | 可在后续实现为打勾、灰化或折叠 |
| all completed | 显示完成反馈 / 进入足迹记录提示 |
