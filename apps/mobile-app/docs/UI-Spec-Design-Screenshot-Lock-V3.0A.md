> 适用范围：`apps/mobile-app/` 手机 App 前端界面  
> 目的：防止 Codex / Claude Code 再把多个设计页合并、重排、自由发挥

## 1. 最高优先级规则

### 1.1 设计稿优先级

本批截图是 V3.0A 手机 App 前端的**视觉验收标准**。

Codex 不允许只根据以下抽象描述自由发挥：

```txt
暖米白背景
深蓝主色
纸张卡片感
温柔低压力
手绘风图标
```

这些只是整体风格，不是页面结构。真正的页面结构以截图为准。

### 1.2 不能再发生的错误

严禁再次出现以下错误：

```txt
把 今日首页 + 任务界面 + 任务执行界面 合并到一个页面
把目标、进度、当前任务、后续任务全部堆在今日页
把页面做成长滚动 Todo List
把任务执行界面做成聊天软件
把旧 Web 页面塞进手机 App 壳
出现双重 AppShell
```

### 1.3 视觉还原要求

实现目标不是“风格相似”，而是：

```txt
页面结构一致
主要文案一致
模块数量一致
信息层级一致
按钮位置一致
页面跳转关系一致
底部 Tab 出现位置一致
不擅自新增模块
不擅自合并页面
不擅自改页面职责
```

## 2. 品牌与文案冲突处理

当前截图中存在以下品牌文案：

```txt
Handy Planner
AI Todo
```

当前项目对外 App 名称锁定为：

```txt
清行
```

因此后续实现前必须由用户确认品牌策略，Codex 不得自行判断。

### 2.1 默认安全策略

在没有用户明确确认前：

```txt
布局、层级、卡片、按钮、间距：严格按截图还原
品牌名称：优先使用“清行”
截图中的 Handy Planner / AI Todo：视为设计稿占位文字，不得自行保留或替换，需用户确认
```

### 2.2 如果用户明确要求“文字也原样”

则按用户最新要求执行，并在执行方案中明确写出：

```txt
本轮允许用户可见 UI 出现 Handy Planner / AI Todo
这是用户确认的视觉还原要求，不是 Codex 自行发挥
```

## 3. 全局视觉锁定

### 3.1 画布与尺寸

```txt
主设计基准：手机竖屏
优先宽度：375–430px
页面形态：App-like，不是网页响应式缩小
底部 Tab：固定在底部，除产品首页 / 登录 / 注册外均显示
```

### 3.2 风格基调

```txt
背景：暖米白 / 轻纸感
主色：深蓝
辅助色：米黄、浅棕、浅灰蓝
卡片：大圆角、柔和阴影、纸张质感
图标：手绘风 SVG
文字：温柔、鼓励、低压力
按钮：大圆角胶囊形，触控友好
```

### 3.3 禁止风格

```txt
禁止强紫色主视觉
禁止强 SaaS 工具风
禁止复杂仪表盘
禁止密集 Todo List
禁止表格 / 数据网格
禁止聊天气泡 UI
禁止普通网页 Landing Page 套壳
```

## 4. 底部 Tab 锁定

手机 App 登录后底部 Tab 固定为：

```txt
今日 / 足迹 / 成长 / 我的
```

禁止：

```txt
改名
新增聊天 Tab
新增项目管理 Tab
新增复杂工具入口
在任务执行界面隐藏底部 Tab
在一个页面里出现两套底部 Tab
```

### 4.1 各 Tab 职责

| Tab | 职责 | 禁止 |
|---|---|---|
| 今日 | 今日首页、生成任务入口、进入任务界面 | 不展示完整历史，不做成长统计 |
| 足迹 | 过去行动记录、空状态、历史任务展开 | 不生成任务，不做今日执行 |
| 成长 | 轻量洞察、完成率、连续天数、建议 | 不做复杂报表 |
| 我的 | 账号、同步、隐私、反馈、版本、退出 | 不做任务执行 |

## 5. 页面流转强锁定

### 5.1 未登录流程

```txt
产品首页
  → 点击“开始使用”
  → 登录页 / 验证码登录页

登录页
  → 密码登录 Tab / 密码登录页
  → 创建行动记录 / 注册页

密码登录页
  → 登录成功
  → 今日首页

注册页
  → 注册成功
  → 今日首页
```

### 5.2 登录后核心流程

```txt
今日首页
  → 输入目标
  → 点击生成任务 / 迈出全新的一步
  → 自动进入任务界面

任务界面
  → 只允许当前任务进入任务执行界面
  → locked 任务点击“陪”只提示，不跳转

任务执行界面
  → 反馈当前任务进展
  → 退出后回到任务界面
  → 完成当前任务后回到任务界面，下一任务成为 current
```

### 5.3 三层页面职责

| 层级 | 页面 | 职责 | 禁止 |
|---|---|---|---|
| 第一层 | 今日首页 | 输入目标、生成任务、开始新一天、轻量今日进度 | 不展示完整任务列表，不展示当前任务卡，不展示后续任务 |
| 第二层 | 任务界面 | 展示生成后的任务清单、当前任务、后续任务 | 不展示目标输入框，不做 AI 反馈输入 |
| 第三层 | 任务执行界面 | 陪用户完成当前任务 | 不展示完整任务列表，不做聊天气泡，不做全局聊天 |


### 禁止

```txt

不能变成今日首页
不能让所有任务都可进入执行界面
不能把 locked 任务做成可自由点击执行
不能做成密集 checkbox Todo List
不能隐藏底部 Tab
不能出现双重 AppShell
```

## 8. Codex 执行硬规则

Codex 在任何前端 UI 任务中必须遵守：

```txt
1. 必须先读取本文档
2. 必须按截图还原页面结构
```
