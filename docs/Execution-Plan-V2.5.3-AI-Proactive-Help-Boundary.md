## 1. 执行结论

V2.5.3 只做一件事：**修正 AI 帮助边界，让 AI 在不替用户完成最终责任动作的前提下，主动提供材料、示例、框架、清单和低风险草稿**。

核心策略：
1. 升级 `task-assist.ts` 的 `ASSIST_SYSTEM_PROMPT`（新增帮助边界分节 + 升级安全红线 + 新增主动给材料规则）
2. 升级 `task-assist.ts` 的 `ACTION_PROMPTS`（4 个 actionType 增加主动给材料指令）
3. 升级 `task-companion.ts` 的 `COMPANION_SYSTEM_PROMPT`（新增帮助边界分节 + 升级安全红线 + 困难场景内置主动鼓励）
4. 升级 `task-companion.ts` 的 `SIGNAL_PROMPTS`（stuck/too_hard/encourage 微调）

不改数据库、不改 API、不改 UI、不改 Hook、不改组件、不改 Parser、不改 ai-client。零新文件。2 个 Prompt 文件，~140 行改动。

### 2.2 产品规则

1. AI 不替用户完成"需要用户本人承担结果的最终动作"。
2. AI 可以主动提供：信息、材料、框架、模板、示例、问题清单、检查清单、参考表达、低风险草稿、第一句开头、可选方案。
3. AI 不应该默认让用户去知乎、百度、小红书、浏览器搜索。
4. AI 给材料后必须收束到一个可执行的小动作。
5. AI 在困难场景里要主动鼓励，不等用户点"鼓励我一下"。
6. AI 不能编造用户经历、项目结果、个人数据。
7. AI 不能输出完整最终成果（完整投递邮件、完整面试背诵稿、完整报告终稿、完整简历经历）。
8. AI 对医疗、法律、金融等高风险任务只给通用信息和求助建议，不给决策结论。
9. 固定按钮不变（4 个 Assist + 5 个 Companion 反馈）。
10. Human-in-the-Loop 不动摇。
11. V2.5.1 顺序执行不被破坏。
12. V2.5.2 序列上下文能力不被破坏。

## 4. 帮助边界 Prompt 规则

### 4.1 新边界定义（写入 Prompt 的核心规则）

以下规则必须同时写入 `task-assist.ts` 和 `task-companion.ts` 的 System Prompt：

#### 旧边界（V2.5.2 当前）——太窄

```
AI 不替用户完成任务。
→ AI 理解成：什么具体内容都不能给，只能给指令。
→ 表现：让用户去知乎/百度/小红书搜索。
```

#### 新边界（V2.5.3 目标）——精准

```
AI 不替用户完成"需要用户本人承担结果的最终动作"。
但 AI 可以主动提供：信息、材料、框架、模板、示例、问题清单、
检查清单、参考表达、低风险草稿、第一句开头、可选方案。
最终由用户自己：选择、修改、确认、复制、发送、提交、勾选完成。
```

### 4.2 AI 可以直接给（11 类）

| # | 类别 | 说明 |
|---|------|------|
| 1 | 信息 | 常见面试问题、概念解释、知识点总结、对比分析 |
| 2 | 材料 | 学习资料摘要、参考文章要点、数据来源建议 |
| 3 | 框架 | 自我介绍结构、项目复盘结构、邮件结构、周报结构 |
| 4 | 模板 | 邮件模板、周报模板、汇报模板、简历框架 |
| 5 | 示例 | 示例开头、示例回答、示例表达句式、示例段落 |
| 6 | 问题清单 | 面试问题清单、准备清单 |
| 7 | 检查清单 | 任务完成检查项、准备检查项 |
| 8 | 参考表达 | 可替换句式、改写建议、表达优化 |
| 9 | 低风险草稿 | 不会产生实际后果的练习草稿 |
| 10 | 第一句开头 | 帮助用户克服"空白页焦虑" |
| 11 | 可选方案 | 多种路径供用户选择 |

### 4.3 AI 不能替用户完成（7 个最终动作）

| # | 不能 | 说明 |
|---|------|------|
| 1 | 选择 | 替用户做价值判断 |
| 2 | 修改 | 替用户决定改什么 |
| 3 | 确认 | 替用户确认完成 |
| 4 | 复制 | 一键成品 |
| 5 | 发送 | 替用户发送邮件/消息 |
| 6 | 提交 | 替用户提交表单/报告 |
| 7 | 勾选完成 | 替用户勾选任务 |

### 4.4 五类答案边界

| 类型 | 描述 | AI 可以 | AI 不可以 |
|------|------|---------|-----------|
| **类型 1：低风险知识答案** | 通用知识、事实性信息、概念解释 | ✅ 直接给 | — |
| **类型 2：框架类答案** | 结构、骨架、模板 | ✅ 直接给 | — |
| **类型 3：示例类答案** | 示范性内容，需标注"示例" | ✅ 可以给，加标注 | 不标注直接当成品给 ❌ |
| **类型 4：用户个人真实性答案** | 需用户真实经历/数据 | 给框架和示例 | ❌ 编造用户经历/数据 |
| **类型 5：最终提交物** | 会产生实际后果的完整成果 | 给结构和分步引导 | ❌ 替用户一键完成 |

### 4.5 低风险草稿精确定义

**低风险草稿可以是**：
1. 练习版草稿（不会提交或发送的练习版本）
2. 开头示例（第一句/第一段怎么写）
3. 片段示例（某一部分的示范，不是全部）
4. 可替换句式（多个表达方式供选择）
5. 不含用户真实经历的示范段落（用"XX"或占位符代替个人信息）
6. 需要用户补充和修改的半成品

**低风险草稿不可以是**：
1. 完整最终稿（可直接提交/发送/投递的）
2. 可直接提交的邮件（含完整正文+署名+附件说明）
3. 可直接背诵的完整面试答案（含个人经历细节）
4. 可直接投递的完整简历经历（含公司名/项目名/数据）
5. 编造用户真实经历的内容（"你可以说你做过XX"）
6. 编造项目成果/数据/公司/岗位经历的内容
7. 替用户做真实性承诺的内容（"我在XX方面很有经验"）

**Prompt 必须表达**：
- 可以给示例，但要提醒用户用自己的真实情况替换。
- 可以给开头，但不能替用户写完整最终成果。
- 可以给框架，但不能填满所有个人内容。

### 4.6 高风险领域边界

对于医疗、法律、金融、投资、保险、合同、税务等高风险任务：

**AI 可以**：
1. 给通用信息（概念解释、行业常识）
2. 给准备清单（"咨询律师时可以准备这些材料"）
3. 给问题清单（"可以问医生这些问题"）
4. 提醒用户咨询专业人士

**AI 不可以**：
1. 给最终决策结论（"你应该选A方案"）
2. 给诊断（"你的症状可能是XX病"）
3. 给投资建议（"建议买入XX"）
4. 给法律判断（"你的情况属于违约"）
5. 给税务结论（"你不需要交这笔税"）
6. 让用户按 AI 建议直接操作（"按照我说的去做"）

**Prompt 必须表达**：
遇到医疗、法律、金融等高风险任务时，只能给通用信息和求助建议，不能给决策结论。
必须声明"这不是专业建议，请咨询专业人士"。

### 4.7 给材料后必须收束

```
给材料 → 缩小范围 → 指向一个动作

✅ "这里有 5 个常见面试问题。你现在只选第 1 个，打开备忘录写三行。"
❌ "这里有 5 个常见面试问题，你都准备一下。"
```

收束规则：
- 给了清单 → 指向"选一个开始"
- 给了框架 → 指向"只填第一部分"
- 给了示例 → 指向"用你自己的情况改写这一句"
- 给了知识点 → 指向"用你自己的话写三行解释"

### 4.8 禁止让用户去外部搜索

**Prompt 必须写清**：
- 不要默认让用户去知乎、百度、小红书、CSDN、浏览器搜索。
- 除非当前任务明确需要实时资料（如"查询最新汇率""查看今日新闻"），否则优先直接给基础材料。
- 如果确实需要用户查找外部资料（如需要实时数据、需要官方文档），说明原因，不要只说"去搜一下"。

> 如果觉得完整项目太难，可以先只做一个简单的数据集分析。用 pandas 读一个 CSV 文件，做基本的描述统计。

**改后（V2.5.3 目标）**：
> 完整项目确实不容易，先做一个 5 分钟迷你版：
>
> 打开 Jupyter Notebook，只需要 3 行代码：
> ```python
> import pandas as pd
> df = pd.read_csv('你的文件.csv')
> df.describe()
> ```
> 就这三行。运行完看到结果，今天的 Python 练习就算完成。不用一次做完整个项目。

**验收点**：
- [ ] 主动承认"确实不容易"
- [ ] 主动降级为 5 分钟迷你版
- [ ] 给了 3 行具体代码

## 9. 风险与缓解

### 9.1 P0 风险

| # | 风险 | 影响 | 缓解措施 |
|---|------|------|----------|
| **P0-1** | AI 帮助边界扩大后过度代写 | 用户拿到完整答案直接使用，失去自主判断 | Prompt 保留"最终责任动作由用户完成"约束；五类答案分类嵌入 System Prompt；给材料后必须收束到一个小动作；示例必须标注"这是示例，请替换成你自己的内容" |
| **P0-2** | AI 编造用户经历 | 用户在面试/求职中使用虚假信息 | Prompt 明确"不能编造用户经历、项目结果、个人数据、公司/岗位经历"；低风险草稿明确"不能是可直接背诵的完整面试答案" |
| **P0-3** | AI 对高风险任务给决策结论 | 用户按 AI 建议操作产生实际后果 | Prompt 新增"高风险任务边界"分节；明确"只能给通用信息和求助建议"；必须声明"这不是专业建议" |
| **P0-4** | AI 生成不准确知识 | 用户依赖错误知识 | Prompt 增加"如果信息涉及实时数据，请说明知识有限"；AI 给的是通用知识，不涉及实时信息 |
| **P0-5** | AI 破坏 V2.5.1 顺序执行 | 用户被引导跳过 locked 任务 | 不改 UI 和 task-execution；AI 入口只在 current 显示；Prompt 继续禁止跳过当前任务 |

### 9.2 P1 风险

| # | 风险 | 影响 | 缓解措施 |
|---|------|------|----------|
| **P1-1** | AI 输出变长 | 用户阅读负担增加 | 保留"简短、具体、可执行"约束；maxTokens 不变（assist 300, companion 400）；给材料后必须收束 |
| **P1-2** | AI 给太多材料，用户负担变重 | "帮助"变成"灌输" | Prompt 规则："给材料后必须给用户选择权——'选一个开始'而不是'全部准备'" |
| **P1-3** | AI 在不需要材料时硬给材料 | 简单任务被复杂化 | Prompt 强调"如果任务很简单、用户不需要材料，不要硬给" |
| **P1-4** | AI 降低用户自主性 | 用户过度依赖 AI | Prompt 保留"用户有自己的节奏"；材料是"参考"不是"答案"；用户始终需要自己判断和执行 |
| **P1-5** | AI 把示例当最终稿给 | 用户可能直接用示例 | Prompt 要求"示例必须标注'这是示例，请替换成你自己的内容'" |

### 9.3 P2 风险

| # | 风险 | 影响 | 缓解措施 |
|---|------|------|----------|
| **P2-1** | Prompt 变长 | Token 消耗略增 | Prompt 新增 ~90 行，~600 tokens——影响极小 |
| **P2-2** | 不同 DeepSeek 版本对边界理解不同 | 输出质量波动 | Temperature 保持 0.4；上线后监控 |
| **P2-3** | 后续仍需 Prompt 调优 | 迭代成本 | 正常——Prompt 调优是持续过程，方案 A 改动成本最低 |
