# Roadmap V3.0B：二级页面与动作闭环

> 状态：规划路线文档  
> 版本：V3.0B  
> 范围：清行 mobile-app 二级页面、轻确认弹层与动作反馈闭环  
> 原则：补齐入口去向，不扩大功能边界，不接真实后端

---

## 1. 背景

V3.0A 已完成清行 mobile-app 的四个底部 Tab 一级页面：

1. 今日 ✅
2. 足迹 ✅
3. 成长 ✅
4. 我的 ✅

V3.0A 的重点是搭好手机端一级信息架构，验证四 Tab 的整体视觉、交互基调与单屏体验。

V3.0B 不是新功能扩张，而是补齐一级页面中已经出现的按钮、入口、二级页面、轻确认弹层和动作反馈，让用户点击后不困惑、不进入死路，也不会误以为某些标签可以点击。

V3.0B 的核心问题是：

- 一级页面里的按钮点了之后去哪里？
- 用户输入后 AI 回应显示在哪里？
- 哪些入口进入二级页面？
- 哪些入口弹出轻确认弹层？
- 哪些入口只是页面内状态切换？
- 哪些看起来像按钮但其实只是标签？

---

## 2. 版本定位

V3.0B 的目标是补齐动作闭环，而不是增加复杂功能。

本版本需要明确：

1. 按钮点了之后去哪里。
2. 用户输入后 AI 回应显示在哪里。
3. 哪些入口跳页面。
4. 哪些入口弹确认。
5. 哪些入口只是页面内状态切换。
6. 哪些内容只是展示标签，不应该点击。

V3.0B 必须继续保持清行当前的产品气质：

1. 手机端优先。
2. 暖米白背景。
3. 深蓝主色。
4. 纸张卡片感。
5. 温柔、安静、低压力。
6. 不做聊天软件。
7. 不做 SaaS 后台。
8. 不做复杂设置中心。
9. 不接真实后端。

V3.0B 只解决“入口与反馈闭环”问题，不进入真实数据、真实账号、真实接口、真实隐私协议或真实反馈系统。

---

## 3. 全局入口分类规则

V3.0B 统一把页面入口分为四类：

### A 类：进入二级页面

适合需要完整阅读、填写或表达的轻页面。

当前适合 A 类的入口：

1. 隐私与数据说明
2. 说说你的想法

A 类入口需要让用户进入一个清晰的二级页面，并提供返回路径。二级页面仍应保留手机端轻量体验，不做长网页，不做设置后台。

### B 类：轻确认弹层

适合可能改变当前状态、让用户感到敏感或需要二次确认的动作。

当前适合 B 类的入口：

1. 退出当前账号
2. 清除本地缓存
3. 开始新的一天，如果会覆盖当前状态

B 类入口不应直接执行敏感动作，应先出现温柔、短文本、低压力的确认弹层。

### C 类：页面内状态切换

适合同一个一级页面内部的流程推进或返回。

当前适合 C 类的入口：

1. 开始任务
2. 换目标
3. 继续陪我推进
4. 回到任务
5. 先退出

C 类入口不需要新增路由，也不需要新增全局页面；应优先在当前 AppShell 内完成状态切换。

### D 类：仅展示标签，不可点击

适合表达洞察、建议、倾向或轻提示的内容。

当前适合 D 类的成长页标签：

1. 更容易开始
2. 小步更稳定
3. 先做 5 分钟
4. 先写一句话
5. 先打开材料
6. 一次只做一个动作
7. 先完成“能开始”的版本

D 类内容必须避免按钮误导：不加箭头、不加点击态、不做 hover/active 反馈，不进入详情页。

## 4. 分阶段路线

V3.0B 拆为四个小阶段：

1. **V3.0B-1：任务执行页 AI 陪伴反馈区**  
   先解决清行区别于普通 Todo 的核心体验：用户输入后，AI 的轻推进建议显示在哪里。

2. **V3.0B-2：我的页二级页面**  
   补齐“隐私与数据说明”和“说说你的想法”两个明显入口的二级页面。

3. **V3.0B-3：我的页轻确认弹层**  
   补齐“清除本地缓存”和“退出当前账号”的轻确认，避免敏感动作直接发生。

4. **V3.0B-4：一级入口总检查**  
   对今日、任务执行、足迹、成长、我的五类入口做收尾检查，避免按钮无去向、死路状态和标签误导。

## 5. V3.0B-1：任务执行页 AI 陪伴反馈区

### 阶段目标

明确用户在任务执行页输入当前情况后，在哪里看到 AI 的轻推进建议。

核心区域名称：

> 给你一个更轻的下一步

该区域用于承接用户反馈后的 AI 轻推进建议，帮助用户从“卡住 / 不知道怎么继续 / 只写了一点”回到一个可执行的小动作。

### 页面结构

任务执行页建议保持以下结构：

```txt
当前任务卡
↓
给你一个更轻的下一步
↓
把现在的情况告诉我
↓
我完成了这一小步
```

### 默认文案

```txt
先不用想完整答案。
你可以告诉我：你做到哪了、卡在哪里，或者只写一句现在的想法。
我会帮你把它变成一个更容易继续的小动作。
```

### 输入后示例

```txt
我看到你已经开始推进了。
现在不用扩展太多，先把其中一个点写清楚就好。

下一小步：
把你刚才提到的内容，写成一句完整的话。
```

### 页面 / 组件范围

V3.0B-1 只围绕任务执行页内部反馈区规划：

1. 当前任务卡。
2. AI 轻推进建议区域。
3. 用户当前情况输入区。
4. “我完成了这一小步”动作。
5. 返回任务相关入口。

### 不做内容

1. 不把该区域做成聊天框。
2. 不把该区域做成输出框。
3. 不做消息流。
4. 不做用户 / AI 对话气泡。
5. 不新增聊天 Tab。
6. 不新增全局 AI 助手入口。
7. 不接真实 AI / API / prompts。
8. 不替用户自动完成任务。

### 验收标准

1. 用户输入后，能明确看到 AI 的轻推进建议显示在“给你一个更轻的下一步”区域。
2. 页面不呈现聊天软件形态。
3. 没有用户 / AI 气泡对话。
4. 没有消息列表无限增长。
5. “我完成了这一小步”仍由用户主动点击。
6. 底部 Tab 保持不变。
7. 不新增全局 AI 入口。
8. 不修改真实后端、API Route、Supabase、数据库或 prompts。

### 是否需要 Claude Code 只读方案

需要。

V3.0B-1 应先由 ChatGPT / 用户确认 UI Spec，再由 Claude Code 做只读架构方案和文件级执行方案。

### 是否需要 Codex 实现

需要，但必须在以下流程完成后才允许：

```txt
UI Spec → Claude Code 只读方案 → ChatGPT 审查 → 用户允许 → Codex 实现 → Claude Review → 用户视觉验收 → 提交
```

## 6. V3.0B-2：我的页二级页面

### 阶段目标

补齐“我的页”中两个明显入口的去向，避免用户点击后无反馈或不知道入口含义。

包含两个二级页面：

1. 隐私与数据说明页
2. 说说你的想法页

### 页面 / 组件范围

V3.0B-2 只围绕“我的页”内二级页面规划：

1. 从“隐私与数据说明”进入隐私说明页。
2. 从“说说你的想法”进入反馈页。
3. 二级页面提供返回“我的页”的路径。
4. 二级页面仍在 mobile-app 的单 AppShell 体验内完成。

### 隐私与数据说明页要求

隐私页应是温柔说明，不是法律长文。

必须满足：

1. 温柔说明。
2. 不写法律长文。
3. 不写复杂技术词。
4. 不出现 Supabase / 数据库 / API。
5. 不吓人。
6. 解释目标、任务、足迹如何帮助用户继续行动。
7. 强调记录是为了帮助用户接着走，不是为了评价用户。

### 说说你的想法页要求

反馈页应是温柔反馈入口，不是客服系统。

必须包含：

1. 温柔反馈入口。
2. 输入框。
3. 提交按钮。
4. 提交后轻提示。
5. 暂不接真实反馈接口。
6. 不叫提交工单。
7. 不叫联系客服。
8. 不做客服系统。

### 不做内容

1. 不写真正隐私协议。
2. 不接真实反馈接口。
3. 不做客服系统。
4. 不出现“提交工单”。
5. 不出现“联系客服”。
6. 不新增真实账号设置。
7. 不接 Supabase / API / 数据库。
8. 不新增路由体系复杂化。
9. 不新增第二套 AppShell。

### 验收标准

1. “隐私与数据说明”有明确去向。
2. “说说你的想法”有明确去向。
3. 二级页面能返回“我的页”。
4. 隐私页温柔、简短、不吓人。
5. 反馈页有输入、提交、提交后轻提示。
6. 反馈提交不接真实接口。
7. 页面不变成 SaaS 设置后台。
8. 不破坏底部 Tab 和 AppShell。

### 是否需要 Claude Code 只读方案

需要。

V3.0B-2 应先由 UI Spec 明确二级页面内容、状态和返回方式，再由 Claude Code 做只读方案。

### 是否需要 Codex 实现

需要，但必须在以下流程完成后才允许：

```txt
UI Spec → Claude Code 只读方案 → ChatGPT 审查 → 用户允许 → Codex 实现 → Claude Review → 用户视觉验收 → 提交
```

## 7. V3.0B-3：我的页轻确认弹层

### 阶段目标

为“我的页”中的敏感动作补齐轻确认弹层，避免用户误触后动作直接发生。

包含两个弹层：

1. 清除本地缓存？
2. 要退出当前账号吗？

### 页面 / 组件范围

V3.0B-3 只围绕“我的页”的轻确认弹层规划：

1. 点击“清除本地缓存”时出现确认弹层。
2. 点击“退出当前账号”时出现确认弹层。
3. 弹层提供取消按钮和确认按钮。
4. 弹层文案保持短、轻、温柔。

### 清除本地缓存弹层

标题：

```txt
清除本地缓存？
```

说明：

```txt
这只会清理这台设备上的临时内容，
不会删除你的账号记录。
```

按钮：

```txt
先不清除 / 确认清除
```

注意：V3.0B 可以只做确认弹层，暂不做真实清理。

### 退出账号弹层

标题：

```txt
要退出当前账号吗？
```

说明：

```txt
退出后，你可以随时再回来。
```

按钮：

```txt
先不退出 / 退出账号
```

点击“退出账号”后：回到欢迎页 / 登录前状态。

### 不做内容

1. 不做真实清缓存。
2. 不删除真实账号记录。
3. 不接真实 Auth。
4. 不改 Supabase。
5. 不改 API Route。
6. 不改数据库。
7. 不做复杂设置确认流程。
8. 不做多级弹窗。

### 验收标准

1. 点击“清除本地缓存”不会直接执行敏感动作，而是先出现确认弹层。
2. 点击“退出当前账号”不会直接退出，而是先出现确认弹层。
3. 取消按钮能关闭弹层并回到“我的页”。
4. 确认退出后回到欢迎页 / 登录前状态。
5. 清缓存在 V3.0B 暂不做真实清理。
6. 弹层不遮挡底部 Tab 的整体体验，不制造复杂设置后台感。

### 是否需要 Claude Code 只读方案

需要。

V3.0B-3 应先明确弹层 UI Spec、按钮文案和状态流，再由 Claude Code 做只读方案。

### 是否需要 Codex 实现

需要，但必须在以下流程完成后才允许：

```txt
UI Spec → Claude Code 只读方案 → ChatGPT 审查 → 用户允许 → Codex 实现 → Claude Review → 用户视觉验收 → 提交
```

成长页暂时不做二级页面。

成长页当前内容是低压力洞察，不是数据报表，也不是行动任务列表。以下内容是成长洞察标签，不是按钮，不跳转：

1. 更容易开始
2. 小步更稳定
3. 先做 5 分钟
4. 先写一句话
5. 先打开材料
6. 一次只做一个动作
7. 先完成“能开始”的版本

后续如果调整成长页，只允许做视觉微调：

1. 降低按钮感。
2. 不加箭头。
3. 不加点击态。
4. 不做详情页。
5. 不做成长报告页。
6. 不做图表页。

成长页必须继续保持：

- 温柔、安静、低压力。
- 不做完成率 / KPI / 百分比。
- 不做评分 / 排名 / 等级。
- 不做 SaaS 仪表盘。

## 11. 推荐执行顺序

### 第一优先级：V3.0B-1 任务执行页 AI 陪伴反馈区

原因：这是清行区别于普通 Todo 的核心体验。用户输入后必须明确看到 AI 的回应在哪里，否则“陪我做”会像一个没有闭环的输入框。

### 第二优先级：V3.0B-2 我的页二级页面

原因：“我的页”已经有两个明显入口：隐私与数据说明、说说你的想法。它们属于用户会自然点击的 A 类入口，需要补齐页面去向。

### 第三优先级：V3.0B-3 我的页轻确认弹层

原因：退出和清缓存属于敏感动作，不应该直接发生。即使 V3.0B 暂不做真实清缓存，也应先补齐确认体验。

### 第四优先级：V3.0B-4 一级入口总检查

原因：这是收尾检查，用来避免按钮无去向、死路状态、误导标签、双重 AppShell 或底部 Tab 被破坏。

## 12. 每阶段流程

### V3.0B-1 / V3.0B-2 / V3.0B-3

这三个阶段都需要完整的产品与工程协作流程：

```txt
UI Spec → Claude Code 只读方案 → ChatGPT 审查 → 用户允许 → Codex 实现 → Claude Review → 用户视觉验收 → 提交
```

说明：

1. UI Spec 先明确页面状态、文案、视觉边界和禁止项。
2. Claude Code 只读方案只做架构判断和文件级执行方案，不直接改代码。
3. ChatGPT 审查后，用户明确允许，Codex 才能实现。
4. Codex 实现后必须经过 Claude Review。
5. 用户视觉验收通过后，才允许提交。

### V3.0B-4

V3.0B-4 优先由 Claude Code 做只读检查：

```txt
Claude Code 只读检查 → ChatGPT 判断 → 如有必要再拆小修任务 → Claude Review → 用户验收
```

V3.0B-4 不一定需要 Codex。如果只读检查没有发现 P0 / P1 问题，可以只形成检查结论，不进入实现。

## 13. 验收总标准

V3.0B 完成时，应满足：

1. 一级页面中的明显按钮都有去向或反馈。
2. 用户输入后，AI 轻推进建议有明确展示区域。
3. “隐私与数据说明”和“说说你的想法”有二级页面。
4. “清除本地缓存”和“退出当前账号”有轻确认弹层。
5. 成长页标签不误导用户以为可以点击。
6. 不存在明显死路状态。
7. 不存在双重 AppShell。
8. 不破坏 BottomTabBar。
9. 不新增聊天 Tab。
10. 不新增全局 AI 助手入口。
11. 不接真实后端。
12. 不修改 API Route、Supabase、数据库 migration 或 prompts。
13. 视觉仍保持清行的暖米白、深蓝、纸张卡片、温柔低压力风格。
